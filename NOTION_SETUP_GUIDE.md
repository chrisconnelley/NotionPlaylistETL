# Notion Database Setup Guide

## Overview

When you update `NOTION_API_KEY` in `.env` to point to a new Notion Teamspace, the app now detects this and automatically offers to set up new databases from the saved schemas.

## Quick Start

1. Update `.env` with your new Notion API key
2. Start the app
3. Click "OK" when prompted to set up databases
4. When asked for a "Parent Page ID":
   - Open **any existing page** in your Notion workspace
   - Copy the page ID from the URL: `https://www.notion.so/{COPY_THIS}`
   - Paste it into the dialog
5. Wait for setup to complete (~10-30 seconds)
6. A new "MusicTunnel" page with all 4 databases will be created inside the parent page

That's it! The databases are fully set up with all properties and relations.

## What Changed

### New Files
- **`notion_config.py`** — Stores Notion database IDs separately from main config. Can be regenerated when switching Teamspaces.
- **`notion/setup.py`** — Functions to create databases from saved schemas and update config.
- **`notion/check_setup.py`** — Functions to detect when you're in a new Teamspace.

### Updated Files
- **`config.py`** — Now imports DB IDs from `notion_config.py` instead of hardcoding.
- **`ui/app.py`** — Added `_check_notion_setup()` to detect new Teamspace on startup and offer to set up databases.

## How It Works

### Step 1: App Startup
When the app starts, it checks if all four databases exist:
```python
check_notion_setup()  # Returns (all_exist: bool, missing: list[str])
```

### Step 2: Detection
If any database is missing, the app:
1. Logs a warning with which databases are missing
2. Shows a dialog asking if you want to set them up
3. If you click OK, asks for a parent page ID

### Step 3: Setup
If you proceed and provide a parent page ID:
1. Creates a new "MusicTunnel" page inside the parent page
2. Creates all 4 databases inside MusicTunnel from the saved schemas:
   - `notion_schema/songs.json`
   - `notion_schema/song_artists.json`
   - `notion_schema/playlists.json`
   - `notion_schema/playlist_songs.json`
3. Sets up relations between databases
4. Updates `notion_config.py` with the new database IDs

### Step 4: Done
The app shows a success message and you can now use the new databases.

## Schemas

The database schemas are stored in `notion_schema/` and are auto-generated by the Console tab's "Verify Schema" button. They include:
- All properties (text, URL, relations, rollups, etc.)
- Property configurations (format, options, etc.)
- Inter-database relations

## Reverting to Old Databases

If you want to switch back to your old databases:
1. Update `.env` with the old `NOTION_API_KEY`
2. Update `notion_config.py` with the old database IDs OR let the app detect and offer setup

## Manual Setup (Alternative)

If automated setup fails or you prefer manual control:
1. Create 4 new databases in your Notion Teamspace
2. Manually copy the property structure from `notion_schema/` to each database
3. Set up the relations manually:
   - **Songs** → `Song Artists` relation (Songs DB)
   - **Playlist Songs** → `Song` relation (to Songs DB)
   - **Playlist Songs** → `Playlist` relation (to Playlists DB)
4. Update `notion_config.py` with the new database IDs

## Troubleshooting

### "Database not found" error on startup
- Your `NOTION_API_KEY` doesn't have access to the database
- Or the database ID in `notion_config.py` is from a different Teamspace
- Try the automated setup flow or manually update `notion_config.py`

### Setup fails with "Provide a `parent.page_id`..."
- Internal Notion integrations can't create databases at workspace level
- The app now automatically creates a "MusicTunnel" page to contain the databases
- Make sure your integration has permission to create pages and databases

### Setup fails with permission error
- Your Notion integration doesn't have permission to create databases
- Go to **Settings → Integrations → Your Integration** and enable **Create pages/databases**
